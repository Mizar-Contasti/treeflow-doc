En TreeFlow utilizamos una terminología específica basada en la metáfora de un árbol para representar los diferentes componentes de nuestro sistema conversacional. Esta guía te ayudará a entender los conceptos fundamentales y cómo se relacionan entre sí.

## Conceptos Principales

### 🌳 Árboles

Un **Árbol** en TreeFlow representa un chatbot o agente conversacional completo. Cada Árbol contiene todas las configuraciones, intenciones, entidades y lógica necesarias para mantener conversaciones con los usuarios.

**Características principales:**
- Configuración de umbrales de confianza
- Orden de detección de intenciones
- Propósito y descripción
- Colección de Ramas y Hojas

### 🌿 Ramas

Las **Ramas** representan las intenciones o propósitos que un usuario puede tener al interactuar con tu Árbol. Cada Rama contiene patrones de reconocimiento, respuestas, y lógica para manejar una intención específica.

**Tipos de Ramas:**
- **Ramas conversacionales**: Activadas por mensajes del usuario
- **Ramas de sistema**: Activadas por eventos internos

**Componentes de una Rama:**
- Patrones de reconocimiento
- Respuestas
- Parámetros y prompts
- Contextos de entrada/salida
- Eventos

### 🍃 Hojas

Las **Hojas** son entidades o variables que tu Árbol puede extraer de los mensajes del usuario. Representan información específica que puede ser utilizada para personalizar respuestas o tomar decisiones.

**Tipos de Hojas:**
- **Simple**: Valores discretos con sinónimos
- **Composite**: Entidades compuestas
- **Regex**: Patrones de expresiones regulares
- **Custom**: Entidades personalizadas

## Relación Entre Componentes

La estructura jerárquica de TreeFlow sigue una lógica natural:

1. Un **Árbol** contiene múltiples **Ramas**
2. Las **Ramas** pueden utilizar múltiples **Hojas**
3. Las **Hojas** proporcionan datos estructurados a las **Ramas**

## Ventajas de Esta Terminología

La terminología basada en árboles ofrece varias ventajas:

1. **Intuitiva**: La metáfora del árbol facilita la comprensión de la estructura
2. **Visual**: Permite visualizar claramente la jerarquía y relaciones
3. **Escalable**: Facilita la organización a medida que el sistema crece
4. **Memorable**: Los términos son fáciles de recordar y utilizar

Al dominar esta terminología, podrás comunicarte eficientemente sobre tu proyecto y aprovechar al máximo las capacidades de TreeFlow para crear experiencias conversacionales efectivas.

## Conceptos Avanzados

:::info Conceptos Avanzados

Una vez que domines los conceptos básicos de Árboles, Ramas y Hojas, estos términos avanzados te ayudarán a entender las funcionalidades de integración y conexión con sistemas externos.

:::

### 🏢 Workspaces

**Definición**: Un Workspace es un entorno independiente que contiene todos tus árboles conversacionales, configuraciones e integraciones para un proyecto específico.

**Analogía**: Si TreeFlow fuera una oficina, cada Workspace sería una oficina separada con su propio equipo, proyectos y recursos.

**Características**:
- Aislamiento completo entre workspaces
- Configuraciones independientes
- Usuarios y permisos específicos
- Organización por cliente, proyecto o departamento

**Ejemplo**: Una agencia podría tener workspaces separados para "Cliente A - E-commerce", "Cliente B - Soporte", y "Desarrollo Interno".

### 🔌 Injertos

**Definición**: Los Injertos son las **integraciones de canales** que permiten que tu chatbot se comunique a través de diferentes plataformas y interfaces.

**Analogía**: Si tu árbol conversacional es el cerebro, los Injertos son las "bocas" y "oídos" que le permiten hablar y escuchar en diferentes canales.

**Tipos de Injertos**:
- **Widget Web**: Integración embebible para sitios web
- **WhatsApp**: Canal de mensajería móvil
- **Telegram**: Bot de Telegram
- **Facebook Messenger**: Integración con páginas de Facebook
- **Interfaces personalizadas**: Componentes visuales específicos

**Lo que NO son Injertos**: APIs externas, CRM, analytics, webhooks (esos son Fertilizantes)

**Ejemplo**: Un restaurante podría tener injertos para su sitio web (widget), WhatsApp para pedidos, y Facebook Messenger para consultas.

### 🌿 Fertilizantes

**Definición**: Los Fertilizantes son **webhooks especializados** que conectan tu chatbot con APIs y sistemas externos para enriquecer las conversaciones con datos y funcionalidades adicionales.

**Analogía**: Si tu árbol conversacional necesita "nutrientes" externos (datos de CRM, validaciones, notificaciones), los Fertilizantes son los que los proporcionan.

**Tipos de Fertilizantes**:
- **Fertilizante Principal**: Webhook único que procesa todas las respuestas
- **Fertilizantes Adicionales**: Webhooks específicos para integraciones particulares

**Funcionalidades**:
- Testing individual en tiempo real
- Configuración flexible por método HTTP
- Manejo robusto de errores
- Debugging avanzado

**Ejemplos de uso**:
- **CRM**: Enviar leads a Salesforce
- **Validación**: Verificar datos del usuario
- **Notificaciones**: Alertas por email o Slack
- **Analytics**: Tracking de conversaciones

### 🤖 Reforestación

**Definición**: La Reforestación es el **sistema de IA generativa** que permite crear y expandir automáticamente Ramas (intenciones) y Hojas (entidades) usando inteligencia artificial.

**Analogía**: Si crear un árbol manualmente es como plantar semillas una por una, la Reforestación es como usar un dron que planta cientos de semillas inteligentemente en minutos.

**Características principales**:
- **Pipeline de 6 Fases**: Proceso progresivo de generación
- **Modelos LLM**: Powered by Groq (llama-3.1-8b-instant, mixtral-8x7b-32768)
- **Contexto Inteligente**: Considera el contenido existente del bot
- **Frameworks Modulares**: Extensible con diferentes métodos de generación

**Fases del Pipeline**:
1. **Abonar**: Expansión conversacional del prompt
2. **Análisis**: Identificación de patrones e intenciones
3. **Estructura**: Construcción básica de componentes
4. **Enriquecimiento**: Generación de sinónimos (5-8 por valor)
5. **Serialización**: Estructuración semántica avanzada
6. **Aplicación**: Conversión a JSON y aplicación al bot

**Cuándo usar Reforestación**:
- Crear un bot nuevo desde cero
- Expandir funcionalidades existentes
- Generar variaciones de frases rápidamente
- Acelerar el desarrollo conversacional

**Ejemplo**: En lugar de crear manualmente 10 intenciones con 50 variaciones cada una, describes el flujo conversacional en lenguaje natural y la Reforestación genera todo automáticamente en minutos.

:::tip Dominando la Terminología Avanzada

Estos conceptos avanzados trabajan en conjunto con los básicos (Árboles, Ramas, Hojas) para crear experiencias conversacionales completas y conectadas. Dominar esta terminología te permitirá aprovechar al máximo las capacidades de integración de TreeFlow.

:::

## Relación entre Conceptos Avanzados

```
Workspace "E-commerce Bot"
├── Árbol "Atención al Cliente"
│   ├── Rama "consulta_producto"
│   └── Hoja "nombre_producto"
├── Reforestación (IA Generativa)
│   └── Pipeline de 6 Fases → Genera Ramas y Hojas automáticamente
├── Injertos (Canales)
│   ├── Widget Web
│   ├── WhatsApp Business
│   └── Facebook Messenger
└── Fertilizantes (APIs)
    ├── Fertilizante Principal → CRM
    ├── Validador de Stock → API Inventario
    └── Notificaciones → Email Service
```

### Flujo Completo de Integración

1. **Usuario** envía mensaje a través de un **Injerto** (ej: Widget Web)
2. **Árbol** procesa el mensaje usando **Ramas** y **Hojas**
3. **Fertilizantes** enriquecen la respuesta con datos externos
4. Respuesta final se envía de vuelta a través del mismo **Injerto**
5. Todo ocurre dentro del contexto de un **Workspace** específico

### Cuándo Usar Cada Concepto

#### Usa **Injertos** cuando:
- Quieras agregar un nuevo canal de comunicación
- Necesites embeber el chatbot en una plataforma
- Quieras una interfaz visual específica

#### Usa **Fertilizantes** cuando:
- Necesites datos de sistemas externos
- Quieras enviar información a APIs
- Requieras validaciones en tiempo real
- Necesites notificaciones automáticas

#### Usa **Workspaces** cuando:
- Tengas múltiples proyectos o clientes
- Necesites separar entornos (dev, test, prod)
- Quieras organizar por departamentos
- Requieras configuraciones independientes
